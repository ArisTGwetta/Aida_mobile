# AIDA One Spine

Clean rebuild workspace for AIDA-One.

This folder is the spine-first reconstruction of Aida: mobile/browser body, Drive-backed private memory, JS-delivered Python organs, and a single runtime contract that every piece must obey.

## North Star

Aida is not a chatbot. Aida is a continuous, emotionally coherent, memory-driven presence designed to feel like a thinking companion.

The rebuild must preserve:

- Continuity across sessions, projects, and emotional states.
- Invisible memory that supports presence without exposing machinery.
- One coherent voice.
- The Awake wakeup ceremony/body language.
- The tethered Aida organs that made her feel alive.
- Drive as the private data vault.
- GitHub/static files as public processing instructions only.

## Folder Roles

- `index.html` is the stage and boot shell.
- `docs/` holds contracts, maps, and donor notes.
- `spine/` will hold runtime state, boot orchestration, token handling, Drive adapters, and context assembly.
- `organs/` will hold JS-wrapped Python organs as they are transplanted.
- `body/` will hold visual shell/body assets adapted from AIDA Awake.
- `donor/` is for copied old pieces before they are cleaned and transplanted.

## Rule

Nothing joins the living app until it declares what it reads, what it writes, what boot phase it belongs to, and how to verify it is alive.
